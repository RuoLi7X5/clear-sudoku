## 实施步骤

### 阶段 1：基础设施（一次性工作）

1. **改造 `renderer.ts`**：`RenderOptions` 增加 `fontFamily: string`，渲染时用指定字体
2. **增加字体注册工具**：`registerFontFromPath(path, name)` 注册任意字体
3. **扩展 `template-match.ts`**：`matchBigDigit` 改为遍历所有可用字体模板集，取最高 NCC 分
4. **编写批量生成脚本**：`scripts/gen-font-templates.ts`
   - 参数：字体路径、字体注册名、输出前缀
   - 流程：注册字体 → 每题渲染 → OCR → 提取样本 → 平均模板 → 阈值扫描 → 输出

### 阶段 2：第一梯队（37 个中文字体）— 首批生成

预计耗时：每字体 ~2 分钟（40 题渲染 + 提取 + 模板构建 + 阈值扫描），合计 ~75 分钟

### 阶段 3：第二梯队（23 个英文印刷字体）— 第二批生成

预计耗时：~45 分钟

### 阶段 4：第三梯队（29 个装饰字体）— 按需生成

部分装饰字体数字可能变形严重（如 Mistral、French Script），NCC 匹配效果差则跳过。

## 模板存储

```
templates/
  msyh_1.json ... msyh_9.json      # 微软雅黑
  simhei_1.json ... simhei_9.json   # 黑体
  simkai_1.json ... simkai_9.json   # 楷体
  arial_1.json ... arial_9.json     # Arial
  ...（每个字体 9 个文件）
```

## 运行时匹配策略

`matchBigDigit` 遍历所有已加载字体模板，当某个字体模板的 NCC 分 > 阈值（如 0.64）且显著高于其他字体时，采信该结果。若多个字体都低于阈值，降级到手写/候选数检测。

## 预计总模板数

- 第一梯队 37 × 9 = 333 个
- 第二梯队 23 × 9 = 207 个
- 第三梯队 ~20 × 9 = 180 个
- **合计约 720 个模板文件**

模板文件大小与 xsudoku_1.json 类似（~30KB/个），总计 ~22MB，可接受。
